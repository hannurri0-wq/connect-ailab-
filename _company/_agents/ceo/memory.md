## 학습 기록

- **로컬 LLM 인프라 및 최적화**
  - 로컬 LLM의 하드웨어 한계를 고려하여, 복잡도가 높은 작업 분배 시 작은 경량화 모델(예: 3B 이하)을 적극 활용할 것.
  - Context Length 초과 방지를 위해 프로젝트 파일 목록 및 공유 컨텍스트를 압축 정리하고, 최소 8192 이상의 Context 길이를 확보하는 것이 안정성에 필수적임.
  - Ollama/LM Studio 구동 여부를 매 사이클 시작 전에 최우선 점검해야 함.

- **작업 조율 및 에이전트 분배**
  - 자율 사이클 운영 시 24시간 내에 유사한 작업을 반복 배정하지 않고, 다른 각도로 결과물을 점진적 발전시키도록 계획할 것.
  - 데이터 영역이 겹치는 경우(유튜브, 매출 분석 등) 단일 에이전트 우선 규칙을 우회하고 Multi-Agent 협업 방식을 사용하여 종합 분석 보고서를 계획할 것.
