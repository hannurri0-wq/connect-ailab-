## 🎯 목표
Mini-Deep Dive 과정 전반(Z Score 측정 → Mini-Preview 진입 → 결제 시도)에서 발생하는 시스템적 오류를 사용자에게 가장 신뢰감 있고, 재시도를 유도하는 방식으로 안내한다. 핵심은 **'실패의 원인을 시스템 문제로 돌려 사용자의 통제감을 유지시키는 것'**이다.

## ⚙️ 기반 기술 사양 (Codari 기반)
*   **오류 처리 기준:** Codari가 정의한 `payment_service.py`의 예외 클래스 및 코드 체계를 최우선으로 참조한다.
*   **톤앤매너:** 네이비/골드, 안정적이고 명료함.

## 🌐 Funnel별 실패 시나리오 매핑

### 1️⃣ Z Score 측정 단계 (Funnel Entry)
| 시나리오 | 발생 원인 (System Error Code) | 사용자 경험 목표 | UI 메시지 가이드라인 | 재시도 CTA 액션 |
| :--- | :--- | :--- | :--- | :--- |
| **[A] 데이터 전송 실패** | `NETWORK_TIMEOUT` | 시스템 문제 인지 및 단순 재시도 유도 | "잠시 네트워크 연결 상태를 확인해 주세요. 데이터를 다시 불러오는 중입니다." (친근한 경고) | [재시도 버튼]: 즉시 API 호출 재실행 |
| **[B] 서버 과부하** | `SERVER_OVERLOAD` | 서비스의 안정성 강조 및 대기 유도 | "현재 트래픽이 많습니다. 잠시 후 다시 시도해 주시거나, 메일 알림을 신청해주세요." (정보 제공) | [알림 받기 버튼]: 이메일/SMS 구독 모달로 연결 |

### 2️⃣ Mini-Preview 결제 시작 지점 (Funnel Mid)
이 단계에서는 금전적 거래가 발생하므로 신뢰도와 명확성이 가장 중요함.

| 시나리오 | 발생 원인 (System Error Code) | 사용자 경험 목표 | UI 메시지 가이드라인 | 재시도 CTA 액션 |
| :--- | :--- | :--- | :--- | :--- |
| **[C] PG 결제 실패** | `PG-AUTH_FAIL` / `CARD_REJECT` | 원인 분석 및 해결책 제시 (사용자 책임 영역) | "결제가 승인되지 않았습니다. 카드사 정책 또는 정보를 확인해 주세요." (명확한 지침 제공) | [다른 결제 수단 시도]: PG 선택 모달 재출현 / [정보 수정]: 결제 정보 입력 폼으로 포커스 이동 |
| **[D] 서버 트랜잭션 실패** | `PAY-UNKNOWN` / `DB_LOCK` | 시스템의 복구 과정 강조 (개발자 관점) | "처리 중 예상치 못한 오류가 발생했습니다. 저희 시스템이 자동으로 문제를 해결하고 있습니다." (기술적 안정감 부여) | [수동 재시도 버튼]: 5초 카운트다운 후 재호출 시도 유도 |

### 3️⃣ 최종 결제 완료 후 리다이렉트 지점 (Funnel Exit)
성공/실패와 관계없이, 시스템의 신뢰를 마지막으로 각인시키는 것이 목표.

| 시나리오 | 발생 원인 (System Error Code) | 사용자 경험 목표 | UI 메시지 가이드라인 | 재시도 CTA 액션 |
| :--- | :--- | :--- | :--- | :--- |
| **[E] 리다이렉트 실패** | `REDIRECT_FAILED` | 단순 기술적 오류로 처리 (최소한의 정보만 제공) | "결과를 불러오는 데 문제가 발생했습니다. 페이지를 새로고침 해주시거나, 고객센터에 문의해주세요." (간결함 유지) | [페이지 새로고침 버튼]: 브라우저 기본 기능 사용 유도 |

## ✨ UI 컴포넌트 디자인 가이드라인 (Design Component Spec)
**[공통 규칙]**
1.  **톤앤매너:** 배경은 미묘하게 어두운 네이비 톤, 경계선과 성공/재시도는 골드 계열을 사용하여 '고급스러우면서도 위험 상황'임을 인지시킨다.
2.  **정보 구조:** [아이콘 (⚠️)] → [제목: 오류 발생] → [본문 설명 (원인 분석)] → [해결책 제시 문구] → [CTA 버튼]의 순서를 유지한다.

**[컴포넌트 Mockup 구성 요소 상세]**
*   **오류 아이콘:** 느낌표(⚠️)를 사용하되, 주변에 미세한 '회복 애니메이션' (예: 펄스 효과)을 주어 시스템이 작동 중임을 암시한다.
*   **CTA 버튼:** 항상 노출되어야 하며, 단순 "재시도" 대신 **"다시 시도하기(Attempt Again)"** 또는 **"다른 방법으로 구매하기(Use Alternative Payment)"**와 같이 구체적인 행동 지침을 포함해야 한다.
