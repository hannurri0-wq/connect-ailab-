## 📌 A. 컴포넌트 1: 퀴즈 결과 페이지 (Quiz Result Page)

**목표:** 사용자에게 '자신의 약점 존재'를 문제 인식(Problem Awareness)시키고, 다음 단계로 이동할 동기를 부여하는 것이 핵심.

| 사용자 액션 | 시스템 반응 및 상태 변화 | UI/UX 명세 및 인터랙션 상세 |
| :--- | :--- | :--- |
| **퀴즈 완료 (Success)** | 1. 최종 점수 계산 API 호출 $\rightarrow$ 2. 약점 ID 추출 $\rightarrow$ 3. 다음 페이지로의 부드러운 전환 시작. | **[성공 메시지]:** "축하합니다! 총 OOO점을 받으셨습니다." 아래에 '핵심 분석 보기' CTA 배치.<br>**[전환 애니메이션]:** 점수와 함께 약점 요약 위젯이 마치 데이터가 로딩되듯이 부드럽게(Ease-out) 나타나야 함. |
| **API 호출 실패 (Error)** | 1. 서버로부터 오류 코드 수신 ($\text{HTTP 500}$). 2. 사용자에게 직접적인 메시지 노출. | **[에러 메시지]:** "데이터 분석 중 일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요."<br>**[재시도 버튼]:** 명확히 표시하며, 클릭 시 로딩 스피너를 돌리되, 3초 간격으로 재시도를 유도하는 가이드를 제공해야 함. |

## 📌 B. 컴포넌트 2: 약점 분석 대시보드 (Weakness Analysis Widget) - 핵심 영역

**목표:** 점수 대신 '진단된 문제'에 집중하게 만들고, 이 문제를 해결하기 위해 리포트 구매로 유도하는 전환 구조 완성.

| 사용자 액션 | 시스템 반응 및 상태 변화 | UI/UX 명세 및 인터랙션 상세 |
| :--- | :--- | :--- |
| **데이터 로딩 (Loading)** | 약점 ID를 기반으로 세부 분석 데이터를 불러오는 중 ($\text{API Call}$). | **[로딩 스피너]:** 단순 점 데이터가 아닌, "현재 사용자님의 아키텍처 취약점을 진단하는 중입니다..." 와 같은 문구와 함께 표시되어야 함. (정보 제공형 로딩) <br>**[지연 시간 처리]:** 4초 이상 지연 시, '네트워크 연결 상태를 확인해 주세요.' 경고 메시지를 추가하여 사용자의 이탈 방지. |
| **약점 발견 (Display)** | Weakness ID와 상세 진단 결과가 성공적으로 표시됨. | **[시각화]:** 단순히 텍스트 나열이 아닌, 'Weakness A' 섹션에 빨간색/주황색 경고 색상 코드를 적용하고, 문제의 원인(예: 전송 계층 부재)을 시각적인 다이어그램으로 제시. <br>**[CTA 위치]:** 진단 결과 하단에 **가장 눈에 띄게 (Primary CTA)** 배치하며, A/B 테스트 가설 기반 문구 사용 ("이 문제를 해결하기 위한 전문가의 액션 플랜이 필요합니다."). |
| **리포트 구매 유도 (Hover)** | '전문 AI 리포트' 버튼 위에 마우스 오버. | **[인터랙션]:** CTA 배경색이 은은하게 밝아지며, 툴팁(Tooltip) 형태로 "문제 해결의 로드맵을 확인하세요"라는 문구를 노출해야 함. (구매 전 심리적 장벽 낮추기). |

## 📌 C. 컴포넌트 3: 리포트 구매 모달 (Purchase Modal & Funnel)**

**목표:** 결제 과정에서 발생하는 모든 예외 상황을 처리하고, 사용자가 안심하며 최종적으로 결제에 도달하게 만듦.

| 사용자 액션 | 시스템 반응 및 상태 변화 | UI/UX 명세 및 인터랙션 상세 |
| :--- | :--- | :--- |
| **결제 정보 입력 (Input)** | 신용카드 정보를 입력하는 중. | **[유효성 검사]:** 필드별로 즉각적인 피드백 제공 (예: 카드 번호 16자리 미만 $\rightarrow$ 빨간색 밑줄 및 "16자리를 채워주세요" 메시지). <br>**[보안 강조]:** 결제 버튼 근처에 SSL 인증 마크나 '안전한 결제 시스템' 문구를 작게 배치하여 신뢰도를 높여야 함. |
| **최종 제출 (Submit)** | 결제 API 호출 시작. | **[로딩 스피너]:** "결제를 처리하는 중입니다. 잠시만 기다려 주세요..." 라는 명확한 안내 메시지를 표시하며, 3초 이상 지연 시 '실패 가능성'을 미리 경고해야 함. |
| **결제 실패 (Failure)** | PG사로부터 결제 거부 또는 오류 코드 수신 ($\text{Payment Failed}$). | **[에러 처리]:** "결제가 완료되지 않았습니다. 다음 중 원인을 확인해 주세요:" 라는 제목으로, **(1) 카드 한도 초과**, **(2) 정보 불일치**, **(3) PG 시스템 오류** 등 명확한 카테고리별 실패 사유와 함께 해결 방법을 제시해야 함. (사용자 이탈 방지 최우선). |
