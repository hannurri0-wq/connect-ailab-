## 🧠 Factoid Quiz Set (총 5개)

### [Quiz #1] 대규모 언어 모델(LLM) 작동 원리
**테마:** 인공지능 및 트랜스포머 아키텍처의 핵심 이해도 측정
**난이도:** 중상 (AI에 관심 있는 개발자 대상)

**(1) 질문:** LLM이 한 단어를 예측하는 과정에서 가장 중요한 역할을 하는 메커니즘은 무엇이며, 이 덕분에 문맥을 파악할 수 있나요?
A. 재귀 신경망(RNN)의 은닉 상태 전파
B. 어텐션 메커니티 (Attention Mechanism)
C. 순환적 피드백 루프를 통한 가중치 업데이트
D. 비선형 활성화 함수(Non-linear Activation Function)

**정답:** B. 어텐션 메커니티 (Attention Mechanism)

**💡 핵심 해설 문구:** 트랜스포머 모델의 핵심인 '어텐션'은 문장 속 모든 단어가 다른 단어를 얼마나 참조해야 하는지를 계산합니다. 이 덕분에 단순히 앞 단어만 기억하는 기존 RNN의 한계를 넘어, 문장의 장거리 의존성(Long-Term Dependency)을 정확하게 포착하여 복잡한 맥락 이해가 가능해졌습니다.

### [Quiz #2] 클라우드 컴퓨팅 아키텍처
**테마:** 분산 시스템 설계 및 확장성을 위한 필수 지식
**난이도:** 중 (대규모 서비스 운영 경험자 대상)

**(1) 질문:** 대용량 트래픽을 처리하는 MSA(Microservice Architecture) 환경에서, 하나의 서비스가 다운되더라도 전체 시스템이 마비되지 않도록 보장하는 설계 패턴은 무엇인가요?
A. 로드 밸런서(Load Balancer)를 통한 분산 요청
B. 캐시-어태치먼트 패턴 (Cache Attachment Pattern)
C. 서킷 브레이커 패턴 (Circuit Breaker Pattern)
D. 메시지 큐잉(Message Queueing)을 활용한 비동기 처리

**정답:** C. 서킷 브레이커 패턴 (Circuit Breaker Pattern)

**💡 핵심 해설 문구:** 서비스 간의 의존성이 높아질수록, 한쪽 서비스가 느려지거나 실패했을 때 연쇄적으로 다른 서비스까지 장애를 일으키는 '전파성 장애(Cascading Failure)' 위험이 있습니다. 서킷 브레이커는 이를 감지하고 일시적으로 호출을 차단하여, 문제가 되는 서비스에 과부하가 걸리는 것을 막고 시스템 전체의 안정성을 확보합니다.

### [Quiz #3] 웹 개발 및 프론트엔드 아키텍처
**테마:** 최신 웹 개발 트렌드와 상태 관리의 중요성
**난이도:** 중 (프론트엔드 실무자 대상)

**(1) 질문:** React나 Vue 같은 SPA(Single Page Application) 환경에서, 컴포넌트들이 필요로 하는 데이터를 중앙에서 일관되게 관리하여 '데이터 불일치'를 방지하는 상태 관리 방식은 무엇인가요?
A. Props Drilling (부모로부터 하위로 props 전달)
B. Context API 사용 및 재렌더링 최적화
C. 전역 상태 저장소(Global State Store, 예: Redux/Zustand) 구축
D. Local Storage를 이용한 영구 데이터 캐싱

**정답:** C. 전역 상태 저장소(Global State Store, 예: Redux/Zustand) 구축

**💡 핵심 해설 문구:** 컴포넌트 트리가 깊어질수록 Props Drilling이 발생하며 유지보수성이 급격히 떨어집니다. 전역 상태 관리 시스템은 애플리케이션의 '단일 진실 공급원(Single Source of Truth)'을 만들어, 어느 컴포넌트에서든 필요한 데이터를 예측 가능하고 체계적으로 가져다 쓸 수 있게 합니다.

### [Quiz #4] 사이버 보안 및 개발자 책임 (DevSecOps)
**테마:** 현대적인 소프트웨어 배포 과정에서의 보안 취약점 방어
**난이도:** 상 (보안에 민감한 시니어 개발자 대상)

**(1) 질문:** 웹 애플리케이션에서 가장 흔하게 발생하는 공격 중 하나로, 사용자가 입력한 데이터가 서버 측에서 적절히 검증되지 않아 SQL 쿼리를 변조하여 데이터를 탈취하는 취약점은 무엇인가요?
A. XSS (Cross-Site Scripting)
B. CSRF (Cross-Site Request Forgery)
C. SQL Injection (SQLi)
D. Denial of Service (DoS)

**정답:** C. SQL Injection (SQLi)

**💡 핵심 해설 문구:** SQLi는 백엔드 개발자가 가장 주의해야 할 보안 위협 중 하나입니다. 사용자의 입력값(Input)을 절대 쿼리 문자열에 직접 연결해서는 안 되며, 반드시 **파라미터화된 쿼리(Parameterized Queries)** 방식을 사용하여 데이터를 코드가 아닌 '데이터'로 취급하도록 분리해야 합니다.

### [Quiz #5] 시스템 설계 및 성능 최적화
**테마:** 데이터베이스 트랜잭션 처리의 이해와 효율성 확보
**난이도:** 중 (백엔드 개발자 대상)

**(1) 질문:** 여러 연산(A, B, C)을 하나의 단위로 묶어 '모두 성공하거나 모두 실패'하도록 보장하는 개념은 무엇이며, 데이터 무결성을 유지하는 데 필수적인가요?
A. 트랜잭션 격리 수준 (Isolation Level) 설정
B. 원자성(Atomicity), 일관성(Consistency), 고립성(Isolation)을 포함하는 ACID 속성
C. 낙관적 동시성 제어 (Optimistic Locking) 사용
D. 비동기 메시지 큐를 통한 최종 일관성 확보

**정답:** B. 원자성(Atomicity), 일관성(Consistency), 고립성(Isolation)을 포함하는 ACID 속성

**💡 핵심 해설 문구:** 트랜잭션의 ACID 속성은 데이터베이스가 신뢰할 수 있는 상태를 유지하기 위한 기본 약속입니다. 특히 '원자성'은 여러 단계의 작업이 마치 하나의 덩어리처럼 처리되어, 중간에 실패하면 처음부터 되돌아가는(Rollback) 메커니즘을 보장한다는 점에서 중요합니다.
