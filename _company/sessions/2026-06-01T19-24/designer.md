### 🎯 1. 핵심 목표 및 전제 조건
*   **목표:** 사용자가 자신의 약점을 '객관적인 데이터'로 인식(Problem Recognition)하고, 이 문제를 해결하기 위한 유료 리포트 구매에 대한 *필요성(Need)*을 느끼게 하는 것.
*   **원칙:** 모든 인터랙션은 **즉각적이고(Immediate)**, **의미를 전달하며(Meaningful)**, **다음 액션을 제안(Action-Oriented)**해야 한다.

### 🚀 2. 사용자 여정별 상세 인터랙션 가이드라인

#### 📍 Stage A: 약점 발견 및 문제 인식 (Problem Recognition)
사용자가 대시보드에서 특정 취약점을 처음 시각적으로 인지하는 순간의 경험.

| 요소 | 이벤트 | 디테일한 인터랙션 (Micro-interaction) | 상태 변화/애니메이션 가이드 |
| :--- | :--- | :--- | :--- |
| **[Weakness Score]** | 데이터 로딩 완료 시 | 단순 숫자 표시가 아닌, 마치 진단기가 작동하는 듯한 애니메이션. | 0%에서 시작하여 속도감 있게 상승 (Progress Bar Fill). 최종 수치에 도달할 때 부드럽게 '멈추는' 효과(Ease-Out Curve) 적용. |
| **[Weakness Keyword]** | 커서 오버 (Hover) 시 | 해당 키워드가 일반 텍스트보다 약간 더 입체적으로 떠오르는 듯한 느낌 부여. | `translateY(-2px)`와 함께 그림자(`box-shadow`)를 미세하게 강화하여 '강조점'임을 인지시킴. |
| **[CTA 버튼]** | 'Mini-Preview 보기' 클릭 직전 | 버튼 자체가 눌리는 애니메이션을 넘어, 일종의 *에너지 축적* 느낌 부여. | 배경색이 1초 동안 어두운 네이비에서 강조 컬러(#FF6B6B)로 깜빡이는 듯한 (Pulsing Glow) 효과 적용 후, 클릭 가능한 상태로 전환. |

#### 📍 Stage B: Mini-Preview 로딩 과정 (The Build-up of Need)
사용자가 'Mini-Preview 보기'를 눌렀을 때 발생하는 모든 대기 시간(Latency) 처리 방안. 이 단계가 가장 중요하며, 공백 시간이 아님을 인지시켜야 합니다.

| 요소 | 이벤트 | 디테일한 인터랙션 (Micro-interaction) | Loading State UI/UX 가이드라인 |
| :--- | :--- | :--- | :--- |
| **[로딩 스피너]** | API 호출 시작 (0~1초) | 일반적인 원형 스피너 사용 금지. '데이터 처리' 또는 '분석 중'을 상징하는 구조화된 애니메이션 사용. | 3개의 데이터 블록(예: [AI Model], [Industry Trend], [User Data])이 순차적으로 점멸하며 로딩됨 (Sequential Flicker). |
| **[텍스트 피드백]** | 로딩 진행 중 | 로딩 스피너 아래에 현재 어떤 작업을 수행하고 있는지 구체적인 텍스트 메시지를 제공. | *예시 1:* "고객님의 Profile을 기반으로 [MSA 아키텍처]의 약점을 교차 검증 중..." / *예시 2:* "전문 리포트 작성을 위해 최신 산업 트렌드 데이터를 컴파일합니다." (매우 구체적이어야 함) |
| **[Mini-Preview 전개** | 로딩 완료 직전 (1.5~2초) | 마치 비밀스러운 정보를 *천천히 공개하는* 듯한 느낌을 부여하여 기대감 극대화. | 내용이 가려져 있다가(Overlay로 덮임), 중앙에서부터 부드럽게 커튼처럼 열리며 핵심 요약본이 드러나는 (Reveal Animation) 효과를 적용한다. |

#### 📍 Stage C: 전문 리포트 구매 CTA 클릭 및 확인
Mini-Preview를 통해 문제 인식이 극대화된 상태에서, 최종적으로 구매 버튼을 누르는 과정.

| 요소 | 이벤트 | 디테일한 인터랙션 (Micro-interaction) | 상태 변화/애니메이션 가이드라인 |
| :--- | :--- | :--- | :--- |
| **[구매 CTA]** | 클릭 순간 | 단순히 색상만 변하는 것이 아니라, '행동을 확정한다'는 무게감을 주는 피드백 필요. | 버튼이 눌리는 동시에 배경색의 명암 대비가 높아지며 (Depth Effect), 미세한 진동 효과(Haptic Feedback 시뮬레이션)를 주어 물리적 상호작용처럼 느끼게 함. |
| **[결제 성공]** | 결제 완료 후 화면 전환 | '성공' 메시지가 단순 팝업으로 뜨는 것을 지양하고, 마치 중요한 *자격증을 취득*한 듯한 느낌을 준다. | 큰 체크 마크와 함께 "축하합니다! [리포트 이름] 구매가 완료되었습니다."라는 문구가 화면 전체를 부드럽게 채우며 (Fade-In), 다음 학습 여정을 안내하는 CTA(예: "내 약점 대시보드에서 확인하기")가 즉각적으로 노출된다. |
| **[결제 실패/에러]** | 결제 오류 발생 시 | 사용자가 당황하지 않도록, 원인을 명확히 설명하고 해결책을 제시해야 함. | 경고색(#D9534F)으로 에러 메시지 영역을 강조하며, "⚠️ 결제 시스템 오류입니다. 잠시 후 다시 시도하거나 다른 방법을 이용해 주세요." 와 같은 **구체적인 다음 행동 지침**을 제공한다. |
