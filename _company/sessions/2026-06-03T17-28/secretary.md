**📌 핵심 목표:** 미조치로 인한 재무적 손실(Financial Loss)을 공포심리로 극대화하여 구매를 유도하는 상태 기계 구축.

---

## I. Danger Red 경고 플로우 및 애니메이션 스펙
*   **Trigger Point (발동 임계점):** 사용자가 필수 입력 값 누락 또는 데이터 주권 위협 시.
*   **경고 유형:** 강력한 애니메이션과 사운드를 통한 즉각적 '충격' 전달.
*   **기술적 요구사항:** 경고 발생 전후의 타이밍(ms 단위)을 정확히 맞추어 사용자의 시선을 고정해야 함. (→ 개발팀 검토 필수)

## II. 핵심 상태 변화 (State Transitions)
| 상태 (State) | Trigger (발동 조건) | Action / Display Logic | 담당 팀 | 비고 |
| :--- | :--- | :--- | :--- | :--- |
| **Initial** | 페이지 진입 시 | $345억 초기 손실액 Danger Red 표시. | Designer/Dev | 필수 공포심리 주입 구간 |
| **Input Missing** | 필수 데이터 누락 감지 | 경고 애니메이션 + "손실 위험" 메시지 오버레이 (0.8 투명도 블랙). | Dev/Designer | Funnel Control 1차 지점 |
| **Risk Identified** | 특정 위협 요소(예: 공급망 취약) 발견 | 구체적인 재무 손실액($X억)을 수치로 강조하는 시각화 컴포넌트 활성화. | Designer/Dev | 가장 중요한 합의 포인트 |
| **Final CTA Lock** | 구매 결정 임계점 도달 | 'Danger Red' 경고가 해제되기 직전, 최종 CTA 오버레이 강제 노출. | Dev/Business | 전환율 극대화 설계 (Flow Control) |

## III. QA 및 테스트 플로우 합의 필요 사항
1.  **에러 시나리오:** 사용자가 고의적으로 필수 정보를 누락했을 때, 시스템이 경고 애니메이션을 몇 초 동안 유지해야 가장 효과적인가? (시간적 정의 필요)
2.  **데이터 바인딩:** '총 예상 손실액' 수치가 실시간으로 계산되고 변하는 로직의 정확한 데이터 연동 포인트를 확정해야 함.
3.  **Fallback Test:** 네트워크 오류 등 외부 요인 발생 시, 공포심리 유도 기능이 마비되지 않고 최소한의 경고를 표시할 수 있는 백업 플로우가 필요한지 검토 필요.
