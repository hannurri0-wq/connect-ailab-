## ✨ Mini-Blueprint Pro: 오류 메시지 컴포넌트 (Failure Handling Component Set)

**디자인 시스템 기반:**
*   **컬러 팔레트:** Primary Navy (#0F2A45), Accent Gold (#B8913D), Error Red (#DC3545 - 최소화 사용), Neutral Gray (#EBEBF5).
*   **폰트:** Noto Sans KR (전문성/신뢰감 강조).
*   **핵심 구조 (Emotion-Driven):** **[공감] → [원인 분석(What)] → [해결책 제시(How to fix)]**

### 📍 접점 1: Z Score 측정 버튼 클릭 실패 시 (초기 진단 단계)

**사용자 심리:** *불안함, 기대감* ("내가 뭘 모르는지 알아봐 줄 거야. 빨리 알려줘.")
**오류 유형:** API Timeout / 네트워크 연결 끊김 (가장 흔하고 가벼운 에러)
**목표:** 당황하지 않게 안심시키고, 재시도를 유도하여 이탈을 막는다.

**[Mockup 상세 디자인: '진단 일시 중단' 팝업/모달]**

| 요소 | 내용 및 지침 | 코드/디자인 포인트 |
| :--- | :--- | :--- |
| **제목 (공감)** | "🚨 잠깐만요. 데이터를 가져오는 데 문제가 발생했습니다." | Navy Background, White Text. 경고 느낌을 주되 강렬하지 않게 디자인. |
| **본문 (원인 분석)** | "현재 저희 시스템이 사용자의 리스크 지표를 측정하는 과정에서 일시적인 통신 오류가 발생했습니다. 데이터 무결성 확보를 위해 재시도가 필요합니다." | 중립적 톤 유지. '시스템'과 '데이터 무결성'이라는 전문 용어 사용으로 신뢰도 강화. |
| **세부 안내 (선택)** | "혹은 인터넷 연결 상태를 확인해 주십시오." | 작은 회색 폰트로 가이드라인 제공. |
| **CTA 버튼 (해결책)** | **[✅ 재측정하기]** (Primary Navy 배경) / [💡 원인 알아보기](Secondary Link, 작게 배치) | 가장 눈에 잘 띄고, 즉각적인 행동을 유도하는 명확한 Primary Button 배치. |

### 📍 접점 2: Mini-Preview 결제 시작 지점 실패 시 (핵심 트랜잭션 단계)

**사용자 심리:** *긴장감, 재정적 불안* ("돈을 쓰려고 하는데 왜 안 되지? 내 돈은 안전해?")
**오류 유형:** Payment Gateway Failure / DB Write Failure (가장 민감한 에러)
**목표:** 사용자의 자산과 진행 상황에 대한 신뢰를 최우선으로 보호하고, 명확한 재시도 경로를 제시한다.

**[Mockup 상세 디자인: '거래 오류 발생' 전용 모달]**

| 요소 | 내용 및 지침 | 코드/디자인 포인트 |
| :--- | :--- | :--- |
| **제목 (공감)** | "💳 결제 처리 중 문제가 발생했습니다." | Navy Background, Gold Accent Border. 돈과 관련된 중요한 단계임을 시각적으로 강조. |
| **본문 (원인 분석)** | "**[핵심 경고]** 현재 계좌/카드 정보 또는 네트워크 환경 문제로 인해 트랜잭션이 완료되지 않았습니다. 고객님의 데이터는 안전하게 보존되어 있습니다." | *굵은 글씨*를 사용하여 '데이터 안전'을 최우선으로 안심시킨다. 실패의 원인을 구체화하여 사용자가 스스로 체크하게 만든다. (e.g., 카드 유효기간, 한도 초과) |
| **세부 안내 (해결책)** | "1. 결제 정보를 다시 확인해주세요. 2. 잠시 후 재시도를 부탁드립니다." | 번호 매기기(Numbered List)를 사용하여 복잡한 지침을 단순화한다. |
| **CTA 버튼 (해결책)** | **[🔄 정보 재확인 및 재시도]** (Primary Navy 배경) / [📞 고객 지원 문의하기](Secondary Link, 즉시 연결) | '재시도'와 '직접 도움 요청'이라는 두 가지 경로를 명확히 분리한다. |

### 📍 접점 3: 최종 결제 완료 후 리다이렉트 실패 시 (최종 확인 단계)

**사용자 심리:** *안도감 → 불확실성* ("결제는 됐나? 화면만 안 바뀌었을 뿐인가?")
**오류 유형:** Server Redirect Failure / Session Expiration (시스템 내부 문제로 인한 오류)
**목표:** 사용자에게 **'진짜 성공했는지'**에 대한 확신을 주고, 만약 문제가 있다면 *어디서* 확인해야 하는지 알려준다.

**[Mockup 상세 디자인: '상태 불명확' 전용 화면]**

| 요소 | 내용 및 지침 | 코드/디자인 포인트 |
| :--- | :--- | :--- |
| **제목 (공감)** | "✅ 결제 처리가 완료되었을 가능성이 높습니다." | Gold Accent, 성공의 느낌을 주되 '확실하지 않다'는 뉘앙스를 함께 전달. |
| **본문 (원인 분석)** | "시스템 내부 통신 오류로 인해 최종 결과 페이지로 연결되지 않았을 수 있습니다. 하지만 결제 시스템 기록상으로는 성공적으로 처리되었습니다." | '가장 중요한 사실(성공)'을 먼저 제시하고, 그 뒤에 '기술적 문제'를 부수적인 원인으로 설명하여 사용자 불안감을 낮춘다. |
| **필수 확인 지침 (해결책)** | "⚠️ 반드시 다음 2곳을 확인해주세요: 1. 이메일로 전송된 [Mini-Blueprint Pro] 영수증 2. 로그인한 계정 대시보드 내 '구매 기록' 섹션" | 체크리스트(Checklist) 형태로 구성하여, 사용자가 취해야 할 행동을 빠뜨리지 않게 유도한다. (가장 중요!) |
| **CTA 버튼 (해결책)** | **[🚀 바로 대시보드로 이동]** (Primary Navy 배경, 가장 크고 확실하게 배치) | 사용자에게 '다음 단계'를 명확히 제시하여 탐색을 계속하도록 만든다. |
